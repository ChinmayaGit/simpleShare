# OCI Deployment Guide for Cloud Drive Pro

Follow these steps to deploy your file sharing application on your Oracle Cloud Ubuntu instance.

## 1. Prepare the Server

Connect to your OCI instance via SSH:
```bash
ssh -i <your-key> ubuntu@<your-instance-ip>
```

Update the system and install Node.js (if not already installed):
```bash
sudo apt update
sudo apt install -y nodejs npm
```

## 2. Deploy the Application

Create a directory for your app and copy your files (`index.html`, `server.js`, `package.json`) into it. Then install dependencies:
```bash
npm install
mkdir uploads
```

Install **PM2** to keep your server running in the background:
```bash
sudo npm install -g pm2
pm2 start server.js --name "fileshare"
pm2 save
pm2 startup
```

## 3. Configure Nginx as a Reverse Proxy

Edit your Nginx configuration:
```bash
sudo micro /etc/nginx/sites-available/default
```

Update the `server` block to serve your `index.html` and proxy `/api` requests to Node.js:

```nginx
server {
    listen 80;
    server_name _;

    root /path/to/your/fileshare; # Update this to your app directory
    index index.html;

    location / {
        try_files $uri $uri/ =404;
    }

    # Proxy API requests to Node.js
    location /api/ {
        proxy_pass http://localhost:3000/api/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }

    # Serve uploaded files
    location /uploads/ {
        proxy_pass http://localhost:3000/uploads/;
    }
}
```

Test and restart Nginx:
```bash
sudo nginx -t
sudo systemctl restart nginx
```

## 4. OCI Security Rules (IMPORTANT)

For other devices to access your site, you must open port 80 (and 443 if using SSL) in two places:

1.  **OCI Security List:**
    *   Go to OCI Console -> Networking -> Virtual Cloud Networks.
    *   Select your VCN -> Security Lists -> Default Security List.
    *   Add an **Ingress Rule**:
        *   Source CIDR: `0.0.0.0/0`
        *   IP Protocol: `TCP`
        *   Destination Port Range: `80`
2.  **Ubuntu Firewall (iptables/ufw):**
    OCI Ubuntu images often have strict default iptables rules. Run these commands:
    ```bash
    sudo ufw allow 80/tcp
    sudo ufw allow 443/tcp
    # Or if using iptables directly:
    sudo iptables -I INPUT 6 -m state --state NEW -p tcp --dport 80 -j ACCEPT
    sudo netfilter-persistent save
    ```

Now you should be able to access your application by visiting `http://<your-instance-ip>` from any device!
